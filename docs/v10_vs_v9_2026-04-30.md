**11 timeouts is the next bottleneck.** Big plans (UCLA Oncology, Acushnet, Stevenson ES, OSI, Bradley) all timed out — same projects that crashed in v9. Model improvement can't help if the CLI doesn't finish.

## What this proves

1. **More data works.** 124 → 153 projects + alias expansion took max recall from 33% to 78%. Not a fluke — 10 projects gained ≥40 points.
2. **Holdouts didn't regress.** No overfitting catastrophe. Sola actually improved on unseen data.
3. **Architecture is fine.** YOLOv8s + the existing pipeline can hit 70%+. We don't need RT-DETR or YOLOv8m.
4. **The bottleneck shifted.** v9's bottleneck was detection. v10's bottleneck is CLI runtime on big plans (pdfplumber memory/timeout) and 4 projects with detections-but-no-tag-match (Imperial, Krispy Kreme — manufacturer-as-tag schedules).

## Next priorities (in order)

1. **Fix the 11 timeouts.** Profile pdfplumber on UCLA/Acushnet/Stevenson — likely page-rendering or per-page OCR runaway. Goal: get all 34 to actually finish.
2. **Imperial / Krispy Kreme schedule parsing.** These have 0% recall but aren't model issues — schedules use manufacturer names as tags. Fix `schedule_parser.py` to handle that pattern.
3. **Promote v10 to production.** Update `takeoff_cli.py` default model to `hvac_yolov8s_v10.pt`. Archive v9.
4. **Plan v11.** Add Label Studio for annotating the 4 holdouts + the 3 `no_detections` projects (Ontario Museum, TVUSD, Lancaster). Aim for ~180-project corpus.
